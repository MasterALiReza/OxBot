- `index.php` is the single webhook/long-poll entry: it loads `botapi.php`, `function.php`, `keyboard.php`, `panels.php`, and `vendor/autoload.php`, validates the Telegram IP, initializes the user session (`user.step`, `Processing_value*`), enforces anti-spam / channel-join / rules, then dispatches on `$text` / `$datain` into state machines.
- Panel provisioning is delegated to `ManagePanel` (from `panels.php`) which internally instantiates one of the adapter classes in `vpn_panel_integrations/` (Marzban, WGDashboard, Hiddify, Marzneshin, MikroTik, S-UI, X-UI, Alireza, IBSng) based on the `type` column of `marzban_panel`; the core bot code never calls panel HTTP APIs directly.
- The same `index.php` also serves two agent sub-bots under `vpnbot/Default/` and `vpnbot/update/`: each has its own `config.php`/`func.php`/`botapi.php`/`keyboard.php` but reuses the parent's `function.php`, `panels.php`, and `vendor/autoload.php` via relative `$Pathfiles` resolution, so all three processes share the same DB schema and adapter layer.
- Multi-tenancy between the main bot and agents is enforced by the `bottype` column (set to `$ApiToken`) on every `user`/`invoice`/`Payment_report` row, so queries are always scoped to the running bot token.
- Admin helpers (`admin.php`, plus `Marzban.php`, `WGDashboard.php`, `hiddify.php`, `marzneshin.php`, `mikrotik.php`, `s_ui.php`, `x-ui_single.php`, `alireza_single.php`, `ibsng.php`) are thin wrappers around the same `ManagePanel` class used by the bot, giving admins the same panel operations as end users.
