Flat PHP MVC-like layout: each top-level `*.php` file is a page controller that loads shared bootstrap from `inc/config.php`, then includes the chrome via `inc/layout_head.php` / `inc/layout_foot.php`. `inc/config.php` centralises session config (14-day lifetime, custom `sessions/` dir), PDO helpers (`db_query/db_fetch/db_fetchAll/db_count`), auth guard (`require_auth()` redirects to `login.php` or returns `HX-Redirect` for HTMX), CSRF guards (`csrf_check_post/get`), flash messages, and locale loading from `text.json` into `$textbotlang`. Pages render HTML directly with inline PHP; there is no template engine. Asynchronous mutations are handled by `ajax/*.php` scripts which re-use `config.php`, enforce `require_auth()` + `csrf_check_post()`, perform DB updates, optionally call bot functions from the parent project's `function.php`, and return either JSON (`agent_auth.php`) or an HTML fragment for HTMX swap (`service_action.php`, `agent_logs_data.php`, etc.). The agent portal (`agent_login.php`, `agent_users.php`, `agent_*.php`) is a separate flow authenticated via Telegram-ID + OTP through `ajax/agent_auth.php`, using its own Bootstrap-based glass UI rather than the main sidebar layout. Static assets live under `css/`, `js/` (Chart.js, htmx.min.js, sortable.min.js, app.js, per-page scripts), and `fonts/vazirmatn/` for Persian typography.
