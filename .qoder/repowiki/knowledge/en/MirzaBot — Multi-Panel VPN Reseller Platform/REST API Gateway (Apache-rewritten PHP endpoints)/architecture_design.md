Flat dispatcher layout: each file under `api/` is a single-resource endpoint (`product.php`, `users.php`, `invoice.php`, `payment.php`, `settings.php`, `category.php`, `discount.php`, `service.php`, `panels.php`, `keyboard.php`, `statbot.php`, `miniapp.php`, `banner_generator.php`, `log.php`). Apache rewrites `/resource` → `/resource.php` via `.htaccess` and forwards the `Authorization` header into `HTTP_AUTHORIZATION`. Every endpoint follows the same bootstrap — `require_once config.php/function.php[botapi.php|panels.php]`, sets UTF-8 headers, then reads `php://input`, decodes JSON, validates a `Token` header against `$APIKEY` plus an on-disk `hash.txt`, sanitizes input with `sanitizeRecursive`, logs the call into `logs_api`, and dispatches on `$data['actions']` through a `switch`. A separate `verify.php` endpoint implements Telegram Mini App init-data HMAC-SHA256 verification (using `WebAppData` as the HMAC key) and returns a session token stored in `user.token`. All DB access goes through shared helpers `select()`, `update()` from `function.php` using PDO; outbound calls go through `telegram()` / `getuser()` / `getuserm()` from `botapi.php` and panel adapters from `panels.php`.
